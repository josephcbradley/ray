# Local PyPI Mirror System

This project provides an automated system for building and hosting a local PyPI mirror. It is designed to support air-gapped or firewalled environments, specifically ensuring that cross-platform (Linux/Windows) and cross-version (Python 3.12, 3.13, 3.14) dependencies—including binary wheels like `psutil` and `debugpy`—are correctly resolved and available.

## Project Structure

- `reqs/`: Place your input requirement files here (e.g., `core.in`, `data.in`, `vis.in`).
- `outputs/`: Autogenerated compiled requirement files for all target platforms and versions.
- `simple/`: The actual PEP 503 compliant repository containing organized wheels and `index.html` files.
- `process_reqs.py`: The master script that compiles, downloads, and indexes the packages.
- `uv-init-local.*`: Wrapper scripts to help developers initialize new projects pre-configured for this mirror.

---

## 1. Setting up Requirements

1.  **Define Core Dependencies:** Edit `reqs/core.in` for packages required by all environments.
2.  **Define Feature Dependencies:** Add other `.in` files to `reqs/` (e.g., `data.in` for pandas/polars).
3.  **Cross-Platform Logic:** The system automatically attempts to compile every file in `reqs/` (except `core.in`) for:
    - **Platforms:** Linux, Windows
    - **Python Versions:** 3.12, 3.13, 3.14

## 2. Building the Mirror

Run the processing script to synchronize the mirror with your requirement files:

```bash
uv run python process_reqs.py
```

**What this does:**
1.  **Compiles:** Uses `uv pip compile` to generate exact pins for every platform/version combination in the `outputs/` folder.
2.  **Consolidates:** Gathers every unique package identified across all output files into `all_pkgs.txt`.
3.  **Downloads:** Uses `pip download` with multi-platform tags (`manylinux_2_34` down to `2_12`) to pull the correct binary wheels for all targets into the `simple/` directory.
4.  **Indexes:** Runs `simple503` to organize the wheels into a PEP 503 structure and generate HTML indexes.

*Errors are logged to `error_log.txt`.*

## 3. Serving the Mirror

To host the mirror for your team, run a server pointing to the project root so that the `/simple/` directory is accessible:

```bash
# Using pypiserver (included in project dependencies)
uv run pypiserver run -p 8080 .
```

The mirror will now be available at `http://<your-server-ip>:8080/simple/`.

## 4. Developer Setup (uv-init-local)

To ensure developers use the local mirror by default and avoid external resolution, use the provided wrapper scripts instead of the standard `uv init`.

### Deployment

**For Linux/macOS:**
```bash
sudo cp uv-init-local.sh /usr/local/bin/uv-local-init
sudo chmod +x /usr/local/bin/uv-local-init
```

**For Windows:**
Add the directory containing `uv-init-local.ps1` to your system `PATH`, or add this function to your PowerShell `$PROFILE`:
```powershell
function uv-local-init { & "C:\path\to\uv-init-local.ps1" @args }
```

### Usage

When starting a new project, simply run:
```bash
uv-local-init my-new-project
```

This will run `uv init` and automatically append the following to the `pyproject.toml`:
- **Local Index:** Sets the default index to your local server.
- **Target Environments:** Pre-configures `uv` to resolve for both Windows and Linux, preventing "locked-in" platform resolution.

## Troubleshooting

- **Missing Wheels:** If a specific package fails to download for Linux, check the `platforms` list in `process_reqs.py`. It currently supports `manylinux` glibc 2.12 through 2.34.
- **Connection Errors:** Ensure the URL in `uv-init-local` scripts matches your server's actual IP/port.
- **Logs:** Always check `error_log.txt` if the build script reports failures.
