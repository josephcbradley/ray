# ray: your local bit of uv

[![Tests](https://github.com/josephcbradley/ray/actions/workflows/test.yml/badge.svg)](https://github.com/josephcbradley/ray/actions/workflows/test.yml) [![Coverage](https://codecov.io/gh/josephcbradley/ray/graph/badge.svg?token=xujs1Jma4v)](https://codecov.io/gh/josephcbradley/ray)

ray is an automated system for building and hosting a local PyPI mirror. It is designed to support firewalled environments.

## Project Structure

- `reqs/`: Place your input requirement files here (e.g., `core.in`, `data.in`, `vis.in`).
- `outputs/`: Autogenerated compiled requirement files for the current target platform and versions.
- `simple/`: The actual PEP 503 compliant repository containing organized wheels and `index.html` files.
- `process_reqs.py`: The master script that compiles, downloads (in parallel), and indexes the packages.
- `ray.sh` / `ray.ps1`: Wrapper scripts to help developers initialize new projects pre-configured for this mirror.

---

## 1. Setting up Requirements

1. **Define Core Dependencies:** Edit `reqs/core.in` for packages required by all environments.
2. **Define Feature Dependencies:** Add other `.in` files to `reqs/` (e.g., `data.in` for pandas/polars).
3. **Platform-Specific Logic:** The system automatically attempts to compile every file in `reqs/` (except `core.in`) for:
    - **Platform:** The platform it is currently being run on (Linux, Windows, or macOS)
    - **Python Versions:** 3.12, 3.13, 3.14

## 2. Building the Mirror

Run the processing script to synchronize the mirror with your requirement files:

```bash
uv run python process_reqs.py sync
```

**Subcommands:**

- `sync`: (Default) Compiles, downloads, and indexes all requirements.
- `compile`: Only generates `.out` files from `.in` files.
- `download`: Downloads wheels from all generated `.out` files.
- `index`: Only rebuilds the PEP 503 structure using `simple503`.

**What this does:**

1. **Compiles:** Uses `uv pip compile` for each requirement file separately (e.g., `core.in` + `ai.in`) to generate exact pins.
2. **Downloads:** Uses `pip download` in parallel for each requirement set, pulling correct binary wheels for the current platform.
4. **Indexes:** Runs `simple503` to organize the wheels into a PEP 503 structure and generate HTML indexes.

*Errors are logged to `error_log.txt`.*

## 3. Serving the Mirror

To host the mirror for your team, run a server pointing to the project root so that the `/simple/` directory is accessible:

```bash
# Using pypiserver (included in project dependencies)
pypi-server run -p 8080 .
```

The mirror will now be available at `http://<your-server-ip>:8080/simple/`.

## 4. Developer Setup (ray init)

To ensure developers use the local mirror by default and avoid external resolution, use the provided wrapper scripts instead of the standard `uv init`.

### Deployment

**For Linux/macOS:**

```bash
sudo cp ray.sh /usr/local/bin/ray
sudo chmod +x /usr/local/bin/ray
```

**For Windows:**
Add the directory containing `ray.ps1` to your system `PATH`, or add this function to your PowerShell `$PROFILE`:

```powershell
function ray { & "C:\path\to\ray.ps1" @args }
```

### Usage

When starting a new project, simply run:

```bash
ray my-new-project
```

This will run `uv init` and automatically append the following to the `pyproject.toml`:

- **Local Index:** Sets the default index to your local ray mirror.
- **Target Environment:** Pre-configures `uv` to resolve for the current platform.

## 5. Offline Python Environments

If you are operating in a completely air-gapped environment, `uv` will not be able to dynamically download new Python interpreters when you run `uv init` or create virtual environments. To solve this, you can pre-cache the necessary Python binaries on an internet-connected machine and transfer them to your offline machines.

### Caching Python Versions

We provide a helper script, `cache_pythons.py`, to safely download specified Python versions to a targeted local directory.

**Note:** `uv` downloads binaries specific to the operating system it is running on. You must run this script on an internet-connected Linux machine to get Linux binaries, and an internet-connected Windows machine to get Windows binaries.

1. **Run the cache script:**

   ```bash
   # Download to a portable directory (e.g., a USB drive or shared folder)
   uv run python cache_pythons.py --dir /media/usb/pythons
   ```

   *By default, this downloads Python 3.12, 3.13, and 3.14. You can pass specific versions as additional arguments.*

2. **Transfer:** Move the downloaded directory to your offline environment.

### Using Cached Pythons Offline

To direct `uv` to use your local cache instead of attempting network downloads, you simply need to point the `UV_PYTHON_INSTALL_DIR` environment variable to your transferred folder.

**On Linux/macOS:**

```bash
export UV_PYTHON_INSTALL_DIR="/path/to/offline/pythons"
```

**On Windows (PowerShell):**

```powershell
$env:UV_PYTHON_INSTALL_DIR="D:\path\to\offline\pythons"
```

Once this environment variable is set, commands like `uv init` and `uv venv` will seamlessly pull from your local Python cache.

## Troubleshooting

- **Missing Wheels:** If a specific package fails to download, check the `platforms` list in `process_reqs.py`. It currently supports `manylinux`, `win_amd64`, and `macosx` tags.
- **Connection Errors:** Ensure the URL in `ray` scripts matches your server's actual IP/port.
- **Logs:** Always check `error_log.txt` if the build script reports failures.
